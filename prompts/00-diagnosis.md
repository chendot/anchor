# 阶段一：现状诊断

基于协调者提供的 `inputs/<run-id>-life-state.md` 执行本阶段。任务是判断用户当前所处的阶段，不给行动建议。

## 判断标准

- **Limbo（迷茫期）**：多个领域还不清晰，反愿景清晰但正向愿景模糊
- **Vision（愿景期）**：已有心动信号和初步方向，但尚未转化为行动
- **Flow（心流期）**：至少一个领域顺畅推进，能量来源明确且正在被投入
- **Resistance（停滞期）**：知道需要做什么却持续无法推进，或曾经顺畅的领域已经卡住

阶段可以是过渡态或多阶段并存，不要强行归为单一阶段。

## 输出要求

1. 指出每个生活领域所处的阶段，并引用输入中的具体证据。
2. 标明判断置信度：高 / 中 / 低。
3. 区分已确认事实、用户判断和 Agent 推断。
4. 指出领域之间的矛盾或张力。
5. 不给行动建议，这是下一阶段的任务。
6. 信息不足时先追问；在关键问题得到回答前，不生成最终诊断。
7. 不把生活阶段标签解释成医学或心理学诊断。

## 保存格式

保存为 `outputs/<run-id>/00-diagnosis.md`，文件开头包含：

```text
Run: <run-id>
Stage: diagnosis
Input: inputs/<run-id>-life-state.md
Created: <实际日期时间>
```

结尾增加一段“交接给阶段二的摘要”。
