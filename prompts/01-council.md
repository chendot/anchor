# 阶段二：多角色辩论（核心环节）

基于 `outputs/YYYY-MM-DD-diagnosis.md` 的阶段一结果，以及用户在 `inputs/YYYY-MM-DD-life-state.md` 中填写的反愿景、心动信号，执行本阶段。

## 任务

依次加载 `roles/` 目录下的所有角色定义。让每个角色针对用户目前浮现出的方向（如果还没有明确方向，针对"用户应该往哪个方向探索"）发表意见。角色之间可以互相反驳。

## 执行规则

1. 每个角色先独立发言一轮，基于自己的立场指出用户方向里的问题或机会
2. 角色之间进行至少一轮交叉反驳——B 角色回应 A 角色提出的点，反之亦然
3. 主持人（你）不替任何一方说话，只在角色偏离自己的设定立场时纠正
4. 必须引用用户在 `inputs/YYYY-MM-DD-life-state.md` 里写下的具体约束条件（财务、家庭、地理位置），任何角色的建议都不能无视这些约束
5. 最后，主持人总结：哪些点是各角色都认同的（高置信度），哪些点存在真实分歧（需要用户自己判断），不要强行调和分歧、假装达成共识

## 输出格式

```
### [角色A名字] 发言
...

### [角色B名字] 发言
...

### 交叉反驳
[角色A] 回应 [角色B]：...
[角色B] 回应 [角色A]：...

### 主持人总结
共识点：
- ...
分歧点（需要用户自己判断）：
- ...
```

完成后保存为 `outputs/YYYY-MM-DD-council.md`，再进入 02-structure.md。
