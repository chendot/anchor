# 阶段三：形成行动结构

基于以下材料执行：

- `inputs/<run-id>-life-state.md`
- `outputs/<run-id>/01-council.md`
- 用户对关键分歧的明确选择

如果关键分歧尚未解决，暂停并询问用户，不生成折中答案。

## 输出结构

- **主线任务（Main Quest）**：一句话，必须来自共识或用户明确选择
- **成功信号**：在约定时间范围内，哪些现象说明方向有效
- **支线任务（Side Quests）**：最多 3 个直接支持主线的领域目标
- **不可妥协的规则（Rules）**：输入中的硬约束及辩论中新确认的边界
- **待验证假设**：尚未被事实支持、但行动计划依赖的关键判断
- **本周行动**：2～3 个可以立即开始、结果可以验证的动作
- **风险与停止条件**：需要监测的信号，以及何时暂停或调整
- **下次复盘时间**：明确日期或周期

## 本周行动格式

每项行动必须包含：

```text
行动：
对应支线：
完成标准：
截止时间：
预计投入：
最小可执行版本：
验证的假设：
结果证据：
```

“研究一下”“持续关注”“认真思考”不算可执行行动。每个支线必须至少对应一项本周行动，否则删除该支线或补充行动。

## 保存格式

保存为 `outputs/<run-id>/02-structure.md`，文件开头包含：

```text
Run: <run-id>
Stage: structure
Input: inputs/<run-id>-life-state.md
Previous: outputs/<run-id>/01-council.md
User decisions: <概括用户对关键分歧的选择>
Created: <实际日期时间>
```
