# 阶段四：复盘

基于以下材料执行：

- `inputs/<review-id>-review.md`
- 上一轮 `outputs/<run-id>/02-structure.md`
- 深度复盘时可补充上一轮诊断和辩论文件

## 周度轻复盘

适用于检查一周行动，不重新讨论整个人生方向：

1. 逐条核对完成标准和结果证据。
2. 将未完成原因区分为投入不足、外部阻碍、优先级改变或假设被证伪。
3. 检查风险信号和停止条件是否触发。
4. 保留、调整或删除下一步行动，并说明原因。
5. 主线没有新证据表明需要改变时，不重新生成主线。

## 月度或季度深复盘

适用于检查方向本身：

1. 完成周度复盘的全部检查。
2. 根据新信息重新执行阶段判断，比较与上一轮的变化。
3. 判断主线变化来自新事实、新约束、假设被证伪，还是执行阻力。
4. 只有主线存在实质疑问时才召集核心角色重新审议。
5. 如需改变主线，明确说明保留什么、放弃什么以及依据。

## 输出要求

- 实际结果与原计划对比
- 已验证或被证伪的假设
- 对主线、支线和行动的处理决定
- 新风险和下一次复盘时间
- 事实、用户判断和 Agent 推断分开陈述

保存为 `outputs/<review-id>/03-review.md`，文件开头包含：

```text
Review: <review-id>
Mode: weekly | deep
Input: inputs/<review-id>-review.md
Previous run: <run-id>
Previous structure: outputs/<run-id>/02-structure.md
Created: <实际日期时间>
```
