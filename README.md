# Anchor 寻锚

一套帮助找到人生方向的 AI Prompt 工作流。核心不是让单一 AI 直接给建议，而是先诊断现状，再通过多个不同立场做压力测试，最后由用户作出选择并落地成可验证的行动。

## 核心逻辑

```text
现状输入 → 阶段诊断 → 多角色辩论 → 用户决策 → 行动结构
                                                   ↓
                       周度轻复盘 ← 执行 ←─────────┘
                              ↓
                         定期深复盘
```

这是一个文档驱动、人工参与决策的工作流，不是自动替用户决定人生方向的软件。关键分歧必须由用户选择，Agent 不能代替用户完成价值判断。

## 目录职责

- `templates/` — 可公开提交的空白输入模板
- `inputs/` — 用户填写的个人输入
- `prompts/` — 入口与各阶段执行规则
- `roles/` — 多角色辩论的立场定义
- `outputs/` — 诊断、辩论、行动结构和复盘结果

`inputs/` 和 `outputs/` 已被 Git 忽略，因此默认不会提交到代码仓库。但 `.gitignore` 不等于端到端隐私保护：使用云端 Agent 时，文件内容仍可能发送给服务提供方，请根据所用工具的隐私政策决定是否提供。

## 开始新一轮

1. 为本轮确定一个 `run-id`，推荐使用 `YYYY-MM-DD`；同一天多次运行可用 `YYYY-MM-DD-a`。
2. 复制 `templates/life-state.md` 为 `inputs/<run-id>-life-state.md` 并填写。
3. 让 Agent 读取 `prompts/run.md`，告诉它“开始新一轮”，并提供输入文件路径。
4. Agent 依次执行诊断和辩论；如有关键分歧，必须暂停并等待用户选择。
5. 用户确认后，Agent 才生成行动结构。

一轮完整输出保存在同一个目录：

```text
outputs/<run-id>/
├── 00-diagnosis.md
├── 01-council.md
└── 02-structure.md
```

## 复盘

复制 `templates/review-input.md` 为 `inputs/<review-id>-review.md`，填写实际进展，并告诉 Agent 上一轮 `02-structure.md` 的路径。

- 周度轻复盘：检查行动是否完成、阻碍和下一步，不重新讨论整个人生方向。
- 月度或季度深复盘：重新检查阶段、主线和约束，必要时再召集角色审议。

复盘结果保存为 `outputs/<review-id>/03-review.md`，并在文件中关联上一轮 `run-id`。

## 角色使用

默认使用四个核心角色：战略顾问、反脆弱审查员、执行现实主义者、价值守门人。新增角色不会自动全部加入；每次最多再选择两个与问题直接相关的角色，避免重复和输出膨胀。

角色可以挑战用户的判断，但必须区分事实、推断和价值选择，并尊重用户明确写下的约束。涉及医疗、法律、投资等专业问题时，本工作流只帮助梳理决策，不替代专业意见。

## 可以自己调整的地方

可以修改输入模板、行动格式和角色定义，但建议保持三条原则：私人数据与公开模板分离、关键分歧由用户决定、所有行动都有可验证的完成标准。
